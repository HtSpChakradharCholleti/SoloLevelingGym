## ✅ REQUIRED - Always Do This

### Variables & Naming
- Use semantic boolean prefixes: `is*`, `has*`, `show*`, `should*`, `disable*`
- Examples: `isLoading`, `showPassword`, `hasRenew`, `shouldUpdate`
- Array names plural: `cards`, `cities`, `plans`
- Loop item names singular: `card`, `city`, `plan`
- Never single letters except loop counters (i, j, k)

### Styling
- NEVER hardcode colors: Use `APP_CONFIG.COLOURS.PRIMARY`
- NEVER hardcode sizes: Use `APP_CONFIG.METRICS.PADDING`
- NEVER hardcode fonts: Use `APP_CONFIG.FONTS.REGULAR`
- Use `StyleSheet.create()` for all styles
- Use `Platform.select()` for OS-specific differences
- Build styles conditionally: `let styles = {...base}; if (cond) styles = {...styles, override};`

### Strings & Text
- NEVER hardcode user-facing text
- Always use: `strings.keyName` from `js_utilities/strings.js`
- Example: `<Text>{strings.saveChanges}</Text>` not `<Text>Save Changes</Text>`

### Components
- Add JSDoc comment to all reusable components
- Add PropTypes and defaultProps
- Use class components (NOT hooks - legacy codebase)
- Include `this._isMounted = false` in constructor
- Set `_isMounted = true` in componentDidMount
- Set `_isMounted = false` in componentWillUnmount
- Export as default: `export default ComponentName;`

### State Management
- Component state for LOCAL UI only: loading, visibility, form input
- Global state for APP-WIDE data: `global.User`, `global.Cards`, `global.theme`
- AsyncStorage for PERSISTENT storage: tokens, cached API data
- Immutable updates: Use map/spread, NEVER direct mutation
- Example: `const updated = array.map(item => item.id === id ? {...item, prop: val} : item);`

### API Calls
```javascript
// REQUIRED PATTERN:
async functionName() {
  try {
    const token = await AsyncStorage.getItem(APP_CONFIG.ASYNC.ACCESS_TOKEN);
    const response = await APIManager.call(API.ENDPOINT_NAME, {
      token,
      param1: value1,
    });
    
    if (response.success) {
      // Handle success
      return true;
    }
    return false;
  } catch (error) {
    console.log(`ERROR: ${API.ENDPOINT_NAME}\n`, error);
    // Fallback to AsyncStorage
    return false;
  }
}
```

### Conditionals
- Use guard clauses (early return): `if (!success) return false;`
- Use ternary for simple decisions: `condition ? value1 : value2`
- Check booleans explicitly: `if (props.disabled === true)` not `if (props.disabled)`
- Check existence: `if (value != null)` or use optional chaining `value?.property`

## ❌ NEVER Do This

1. Hardcode colors/fonts/sizes anywhere
2. Hardcode user-facing text (use strings.js)
3. Use `useState` hooks (class components only)
4. Create separate component variants (use props for variations)
5. Use Redux, MobX, Context API (use global + AsyncStorage)
6. Mutate state directly: No `this.state.array.push()` or `obj.prop = value`
7. Skip PropTypes on reusable components
8. Skip error handling on API calls
9. Inline styles in production (use StyleSheet.create)
10. Skip try/catch on async operations

## 📁 File Organization

### Place components in:
- `components/buttons/` - Button, TextButton
- `components/inputs/` - Input, DatePicker, Picker, etc.
- `components/text/` - Label, Title, Description
- `components/lists/` - List containers and items
- `components/modules/` - Complex composites (with logic)
- `components/widgets/` - Data visualization
- `components/navigation/` - Navigation UI
- `components/wrappers/` - Layout wrappers (like Wrapper.js)

### Place screens in:
- `views/[feature]/` - All screens for feature together
- Examples: `views/account/`, `views/parking/`, `views/transit/`
- Screen file names: PascalCase with optional underscores
- Examples: `Dashboard.js`, `Account_Main.js`, `Park_FindParking.js`

### Utilities in `js_utilities/`:
- `AppConfig.js` - Design tokens (colors, fonts, sizes, styles)
- `DataManager.js` - Data fetching & initialization
- `APIManager.js` - HTTP requests
- `Helpers.js` - Utility functions
- `strings.js` - i18n text
- `ValidateInputs.js` - Form validation

## 💡 Common Patterns

### API Call with Fallback
```javascript
async getdata() {
  try {
    const token = await AsyncStorage.getItem(APP_CONFIG.ASYNC.ACCESS_TOKEN);
    const response = await APIManager.call(API.GET_DATA, { token });
    if (response.success) {
      global.Data = response.data;
      return true;
    }
  } catch (error) {
    console.log(`ERROR: ${API.GET_DATA}\n`, error);
  }
  // Fallback to cached
  const cached = await AsyncStorage.getItem(APP_CONFIG.ASYNC.DATA_CACHE);
  if (cached) global.Data = JSON.parse(cached);
  return false;
}
```

### Immutable State Update
```javascript
const updatedCards = this.state.cards.map((card) => {
  if (card.id === cardId) {
    return { ...card, is_default: true };
  }
  return { ...card, is_default: false };
});
this.setState({ cards: updatedCards });
```

### Conditional Styling
```javascript
let buttonStyles = APP_CONFIG.STYLES.buttonWrapper;
if (props.danger) {
  buttonStyles = { ...buttonStyles, backgroundColor: APP_CONFIG.COLOURS.DANGER };
}
if (props.disabled) {
  buttonStyles = { ...buttonStyles, backgroundColor: APP_CONFIG.COLOURS.LIGHTGRAY };
}
```
